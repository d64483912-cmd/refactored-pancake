## Summary
This PR introduces an in-memory session management foundation for automation workflows, enabling unique UUID-based sessions accessible via `/new/[UUID]`. It adds database tables, typed backend routes, and frontend types to support creating, listing, and fetching sessions and posting messages.

## Changes
- Database: Add tables
  - `automation_sessions` (session metadata and status)
  - `session_messages` (chat history per session)
  - `session_integrations` (connected services per session)
  - `session_automations` (generated code/scripts per session)
- Backend API (Hono): new `session-routes`
  - `POST /api/sessions` — create a session (optional initial message)
  - `GET /api/sessions/:sessionId` — fetch session details + messages/integrations/automations
  - `GET /api/sessions` — list user sessions (sorted by `updatedAt` desc)
  - `POST /api/sessions/:sessionId/messages` — append a message to a session
- Routes are protected with `authenticatedOnly` and typed via `HonoContext`
- Frontend: add `src/types/session.ts` for type-safe usage
- Drizzle: generate migration `0001_busy_texas_twister.sql`

## Impact
- Establishes a type-safe foundation for automation-session chat UX and agent integrations
- Enables UI work for `/new/:sessionId` and chat thread rendering using existing `SimpleChat`
- Clean separation of concerns between auth, sessions, and AI routes
